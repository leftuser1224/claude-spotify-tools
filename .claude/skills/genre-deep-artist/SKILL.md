# genre-deep-artist

ユーザーがすでによく聴いているジャンルの中で、まだ出会っていないアーティストを掘り起こす。
「好きなゾーンをもっと深く」という探索欲に応えるスキル。

## 手順

### 1. プロファイルと変化を把握（並行実行）

- `build_taste_profile(time_range="long")` — ジャンル比率（%付き）とアーティスト一覧
- `analyze_listening_evolution()` — consistent/rising/fadingリストを把握（除外用）
- `get_top_artists(time_range="long", limit=20)` — ランダム抽出用の20人プール

### 2. 深掘りするジャンルを選ぶ

`genre_breakdown` の `weight_pct` 上位3〜5ジャンルが対象候補。

プロンプトにジャンル指定があればそれを優先。なければ `weight_pct` が最も高いジャンルを選ぶ。

### 3. 複数ソースで未踏アーティストを探す（並行実行）

以下3つを**同時に**実行し、候補プールを作る：

**① グラフ探索**：`get_top_artists` の20人の中から選んだジャンル関連アーティストを **ランダムに5人抽出** し並行実行：
```
traverse_artist_graph(artist_name=<random_artist_1>, hops=1, branch=4)
...×5
```

**② タグ上位**：
```
get_tag_top_artists(tag=<深掘りジャンル>)
```

**③ 隠れた候補**：
```
find_hidden_gems(min_listeners=50000, max_listeners=500000)
```

3ソースの結果を合算し、除外リストを適用した残りを候補プールとする。

### 4. 候補を検証する

候補アーティストに `get_artist_top_tags` を呼び、選んだジャンルのタグが含まれていることを確認する。
ジャンルから外れているものは除外し、ジャンル内に留まる候補を優先する。

### 5. 除外ルール

以下はすべて推薦しない：
- `analyze_listening_evolution` の consistent / rising / fading に含まれるアーティスト
- `get_top_artists` で取得した50人に含まれるアーティスト

### 6. 出力フォーマット

```
## [ジャンル名]をもっと深く

あなたがよく聴く **[ジャンル名]** の中で、まだ出会っていないアーティストを選びました。

1. **[アーティスト名]**
   ジャンル: [タグ]
   → [なぜこのジャンルの文脈で合うか1〜2文。既知のアーティストとの類似点や差異を具体的に]

...（3〜5人）

---
気になる人がいたら流します。アルバム一覧を見ることもできます。
```

### 7. 再生オファー

- 選ばれたら `search_artists` でURIを確認 → `play_context` でアーティスト再生
- 「アルバム見たい」なら `search_artists` でID取得 → `get_artist_albums`
- 「もっとマイナーなのが聴きたい」→ `find_hidden_gems` で知名度を絞って再探索

## 注意点

- **ジャンルの外に出ない**。genre-adjacent と違い、隣接ジャンルへの誘導は不要
- 「既知アーティストに似すぎる」候補は避け、同ジャンル内でも少し新鮮さのあるものを選ぶ
- ユーザーが「もっと違う感じも聴いてみたい」と言ったら genre-adjacent を提案する

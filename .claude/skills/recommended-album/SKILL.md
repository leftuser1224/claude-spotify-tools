# recommended-albums

Spotify聴取データ × Last.fm音楽グラフを使って、**聴取履歴にない・かつ音楽的に近いアルバム**を見つける。アーティスト発見 → そのアルバムへ、という2段階で精度高く掘る。

## 手順

### 1. 好みの把握 + 除外リスト作成（並行実行）

- `build_taste_profile(time_range="long")` — ジャンル比率（%付き）
- `get_top_tracks(time_range="long", limit=50)` — アルバム名を収集して「既聴アルバム除外リスト」を作る
- `get_recently_played(limit=50)` — 直近のアルバムも除外リストへ
- `analyze_listening_evolution()` — `consistent`（常連アーティスト）把握＋深掘り枠の候補リスト用

### 2. 好みの軸を読み取る

`build_taste_profile` の `genre_breakdown` から主要ジャンル・地域・テクスチャを把握する。アルバム探しのキーワードとして使える形に整理する（例：「2010s UKインディーロック、ざらついたギター感」）。

### 3. 未探索アルバムを探す

**アプローチA：アーティスト発見 → アルバム取得（メイン）**

1. `find_hidden_gems(min_listeners=<適切な下限>, max_listeners=<適切な上限>, limit=10, time_range="long")` で未聴アーティストを取得
2. 上位3〜5人の `search_artists` でIDを取得 → `get_artist_albums(artist_id, limit=5)` でアルバム一覧
3. 除外リストにないアルバムを候補に

**アプローチB：直接検索（補助）**

ジャンル・地域・年代のキーワードで `search_albums` を1〜2クエリ実行：
- 例：`"uk indie rock 2020s"` / `"japanese alternative 2024"` / `"shoegaze album"`

### 4. 深掘り枠（常連アーティストの未聴アルバム）

`analyze_listening_evolution` の `consistent` アーティストに未聴アルバムがある場合は「深掘り枠」として別枠で提示する。常連アーティストをよく聴いているなら、まだ聴いていないアルバムの方が発見感がある。

`get_artist_albums` でアルバム一覧を取得し、除外リストと照合して未聴のものを探す。

### 5. 出力フォーマット

```
## おすすめアルバム（未探索枠）

1. **[アルバム名]** / [アーティスト]（[年]）
   Last.fm リスナー: [数]人 / ジャンル: [タグ]
   → [なぜこのアルバムか：ジャンル・地域・規模感が近い理由を1〜2文で]

...（3〜5枚）

---
### 深掘り枠（よく聴くアーティストの未聴アルバム）
- **[アルバム名]** / [アーティスト]（[年]）— [一言]

流しますか？番号を教えてください。
```

### 6. 再生

- 選ばれたら `search_albums("[アルバム名] [アーティスト名]")` でURIを確認 → `play_context(uri)` で即再生

## ポイント

- ベスト盤より通常盤を優先（アルバム単位で聴ける完成度の作品）
- 同じアーティストの複数アルバムを並べない（多様性を持たせる）
- 除外リストのアルバムは絶対に推薦しない
- 有名アーティストも対象に含めてよい（上限なし）
- プロンプトに気分・シチュエーション（「夜に聴きたい」「ドライブ用」等）があれば最優先の絞り込み条件にする
- `get_album_info(artist, album)` でLast.fmタグ・再生数を確認してアルバムの色を補強できる

# listener-profile

過去1年以上のSpotifyデータ × Last.fm音楽グラフを使って、「このリスナーはどんな人か」を多角的に描き出す。ジャンルの好み・開拓傾向・時間帯別ムード・変遷の4軸で立体的なポートレートを作る。

## 手順

### 1. データ収集（並行実行）

- `build_taste_profile(time_range="long")` — 音楽DNA（ジャンル比率・上位アーティスト × Last.fmタグ）
- `analyze_discovery_rate()` — 新規開拓スコアとリスナータイプ診断
- `analyze_mood_by_time()` — 朝/昼/夕/深夜の時間帯別ムードプロファイル
- `analyze_listening_evolution()` — 短期→長期のジャンル変遷とアーティストの動き

### 2. 4軸の読み取り

**軸1：音楽DNA（何を聴くか）**
- `genre_breakdown` から上位ジャンルを束ねてマクロジャンルに整理
- 地域（UK / 日本 / US等）・年代感（タグから推測）を付加
- 「このリスナーの音楽的ルーツ」として言語化する

**軸2：開拓スタイル（どう聴くか）**
- `discovery_score` を解釈する
  - 70以上 → 常に新しいものを探している冒険家
  - 40〜70 → バランスよく新旧を聴くExplorer
  - 40未満 → 信頼できる好みを深く聴くComfortable型
- `new_artists_recently` を見て「最近どこに向かっているか」を補足

**軸3：時間帯と気分（いつ聴くか）**
- `mood_by_time` から各時間帯のトップタグを読む
- 朝と夜で傾向が違う場合は「音楽で気分を切り替えている」と解釈
- 特定の時間帯に集中している場合はその傾向も言及

**軸4：変遷（どう変わってきたか）**
- `genre_by_period` の long→short の差分から「音楽的成長の方向」を読む
- `rising` アーティストから「今どこに向かっているか」を読み取る
- 変化が小さければ「音楽の軸が確立している」と伝える

### 3. 出力フォーマット

```
## あなたの音楽リスナープロファイル

### 音楽DNA
[マクロジャンルの説明。地域・年代・テクスチャを含む2〜3文]

主なジャンル構成：
- [ジャンル]：XX%（[一言説明]）
- [ジャンル]：XX%
- [ジャンル]：XX%

核となるアーティスト（常連）：[consistent アーティスト列挙]

---

### 聴き方のスタイル
**[リスナータイプ名]**（開拓スコア：XX/100）
[discovery_score の意味を自分の言葉で解釈した1〜2文]

最近新たに聴き始めたアーティスト：[new_artists_recently から抜粋]

---

### 時間帯と気分
| 時間帯 | 主なジャンル | 読み取れる気分 |
|---|---|---|
| 朝（5〜12時） | [タグ] | [解釈] |
| 昼（12〜17時） | [タグ] | [解釈] |
| 夜（17〜22時） | [タグ] | [解釈] |
| 深夜（22〜5時） | [タグ] | [解釈] |
※ 再生が少ない時間帯は「データ少なめ」と記す

---

### 変遷：今どこへ向かっているか
[analyze_listening_evolution から読み取ったジャンルシフトと方向性を2〜3文で]

**最近浮上してきたアーティスト（rising）：** [列挙]
**一段落したアーティスト（fading）：** [列挙]

---

### 一言でいうと
[このリスナーを1文で表現するキャッチコピー]
例：「UKインディーとJロックを行き来しながら、静かに新しい音楽を探し続けるExplorer」
```

## ポイント

- データを読み上げるのではなく、**「このリスナーはどんな人か」を語る**ように解釈する
- キャッチコピーは抽象的すぎず、ジャンル・スタイル・傾向を具体的に盛り込む
- 時間帯データが少ない（track_count が5以下）スロットは無理に解釈せず「データ少なめ」と記す
- プロンプトに「〇〇だと思うんだけど合ってる？」などの仮説があれば、データで検証してから答える

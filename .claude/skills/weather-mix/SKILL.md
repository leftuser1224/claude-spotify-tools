# weather-playlist

場所の現在天気と時間帯からムードに合うアーティストを選び、関連アーティストも含めたプライベートプレイリストを作成して再生する。

## 手順

### 1. 場所を確認する

プロンプトに場所が含まれていれば使用。なければユーザーに「今どこにいますか？」と聞く。
都市名なら英語・日本語どちらでもよい（例: 東京、Tokyo、Osaka、New York）。

---

### 2. 天気と現地時刻を取得する

`get_weather` と `get_time` を**同時に**呼ぶ：

```
get_weather(location="{location}")
get_time(location="{location}")
```

それぞれのレスポンスから以下を読み取る：
- `get_weather` → `weather`（天気）、`temp_c`（気温℃）
- `get_time` → `datetime`（ISO 8601形式の現地日時。例: "2026-06-06T14:30:00+09:00"）

時間帯の判定は `get_time` の `datetime` から時刻部分を使う。`get_time` が失敗した場合は `get_weather` の `local_time` にフォールバックする。

どちらも失敗した場合（`success: false`）：ユーザーに天気と現在時刻を直接聞き、Step 3 へ進む。

---

### 3. ムードとシードアーティストを決める

天気・時間帯・季節・気温の4軸からムードを判定し、そのムードに合うシードアーティストを **4〜6組** 選ぶ。

#### 時間帯の区分
| 区分 | 時間 |
|---|---|
| morning | 5〜10時 |
| midday | 10〜14時 |
| afternoon | 14〜18時 |
| evening | 18〜21時 |
| night | 21〜24時 |
| late night | 0〜5時 |

#### 天気グループ
| 天気の説明 | グループ |
|---|---|
| Sunny / Clear / Mostly sunny | sunny |
| Partly cloudy / Overcast / Cloudy | cloudy |
| Light rain / Rain / Drizzle / Showers | rainy |
| Snow / Blizzard / Sleet | snowy |
| Fog / Mist | foggy |
| Thunderstorm / Heavy rain | stormy |

#### 季節の決め方（北半球基準）
| 月 | 北半球 | 南半球 |
|---|---|---|
| 3〜5月 | spring | autumn |
| 6〜8月 | summer | winter |
| 9〜11月 | autumn | spring |
| 12〜2月 | winter | summer |

#### ムード × アーティスト選定の例

組み合わせの参考例（固定ではなく状況に応じて最適なアーティストを選ぶ）：

| ムード | 例となるアーティスト傾向 |
|---|---|
| 晴れ・夏・昼 | アップテンポ、爽やか、夏感（例: Pharrell Williams, SIRUP, kZm） |
| 曇り・夏・深夜 | ambient, lofi, jazz hip-hop（例: Nujabes, Tycho, Sable） |
| 雨・秋・夜 | introspective, indie, post-rock（例: Bon Iver, toe, The National） |
| 晴れ・春・朝 | 軽やか, acoustic（例: Jack Johnson, あいみょん, Vaundy） |
| 霧・冬・夕方 | ambient, melancholic（例: Ólafur Arnalds, Nils Frahm,坂本龍一） |
| 雨・夏・午後 | R&B, neo soul, citypop（例: Frank Ocean, cero, 宇多田ヒカル） |

アーティストは国内外をバランスよく混ぜてよい。ユーザーの地域（日本なら邦楽も自然に含める）を考慮する。

ユーザーへの提示フォーマット：

```
📍 [場所] ／ 🌦️ [天気]、[気温]°C ／ 🕐 [時刻]

**今夜の気分:** [ムードを一言で。例：「夏の深夜、薄曇りの静けさ」]

選んだシードアーティスト:
- [アーティスト名]（[選んだ理由を一言]）
- ...
```

---

### 4. 関連アーティストを探す

シードアーティストそれぞれについて **並行して** `get_similar_artists` を呼ぶ：

```
get_similar_artists(artist_name="<アーティスト1>")
get_similar_artists(artist_name="<アーティスト2>")
...
```

各結果から上位2〜3組を抽出し、シードと合わせて **計10〜15組** のアーティストリストを作る。重複は除く。

---

### 5. 各アーティストのトップトラックを収集する

全アーティスト（シード＋関連）について **並行して** トップトラックを検索する。
`search_tracks` を使い、アーティスト名で検索して代表曲を取得：

```
search_tracks(query="artist:<アーティスト名>", limit=3)
```

各アーティストから **2〜3曲** を選び、合計 **25〜35曲** を目安に集める。

---

### 6. プライベートプレイリストを作成してトラックを追加する

プレイリスト名はムードと場所・時間帯を反映したものにする。

```
create_playlist(
  name="[場所] [時間帯] — [ムード一言]",
  description="[天気]、[気温]°C、[時刻] の[場所]に合わせて生成",
  public=False
)
```

作成後、収集したトラックのURIリストを `add_tracks_to_playlist` で追加する：

```
add_tracks_to_playlist(playlist_id="<id>", track_uris=["spotify:track:xxx", ...])
```

---

### 7. 再生する

```
play_context(uri="spotify:playlist:<playlist_id>")
```

再生後、以下のフォーマットでユーザーに報告する：

```
## 🎵 [プレイリスト名]

📍 [場所] ／ 🌦️ [天気]、[気温]°C ／ 🕐 [時刻]

[N]曲 • プライベートプレイリスト

**シードアーティスト:** [アーティスト名をカンマ区切り]
**関連アーティスト:** [アーティスト名をカンマ区切り]

再生中です。「もっとアップテンポに」「邦楽だけに絞って」などリクエストOKです。
```

---

## 追加要望への対応

- **「もっと静かなの」「激しめに」**: シードアーティストを変えて Step 4 からやり直し、新しいプレイリストを作成
- **「日本語の曲で」**: 邦楽アーティストのみに絞ってやり直し
- **「別のにして」**: シードを変えて新プレイリストを作成
- **「保存して」**: すでにプライベートプレイリストとして保存済みなのでその旨を伝える

## 注意点

- `get_time` は `get_weather` より正確な現地時刻を返すので時間帯判定に優先して使う
- プレイリストは必ず `public=False`（プライベート）で作成する
- `get_similar_artists` が失敗したアーティストはスキップしてよい
- トラック収集時に重複するトラックは除外する
- アーティスト名は英語表記でAPIに渡す（日本語アーティストも英語ローマ字/英語名で）

# genre-explorer-artist

ユーザーの趣味プロファイルから最も遠いジャンルへ飛び込み、それでも「これは面白い」と思えるアーティストを見つける。
ジャンルの壁を越える冒険的な探索スキル。

## 手順

### 1. プロファイルを把握（並行実行）

- `build_taste_profile(time_range="long")` — 支配的ジャンルとアーティスト一覧
- `analyze_listening_evolution()` — consistent/rising/fadingリストを把握（除外用）
- `get_top_artists(time_range="long", limit=50)` — ランダム抽出用の50人プール

### 2. 「対極ジャンル」を選ぶ

まず `artist_profiles` の上位50人の中から **ランダムに5人抽出** し、**並行で** `traverse_artist_graph` を走らせてユーザーの音楽グラフの「外縁」を把握する：

```
traverse_artist_graph(artist_name=<random_artist_1>, hops=2, branch=4)
traverse_artist_graph(artist_name=<random_artist_2>, hops=2, branch=4)
traverse_artist_graph(artist_name=<random_artist_3>, hops=2, branch=4)
traverse_artist_graph(artist_name=<random_artist_4>, hops=2, branch=4)
traverse_artist_graph(artist_name=<random_artist_5>, hops=2, branch=4)
```

depth=2 で発見されたアーティストの `get_artist_top_tags` を収集し、グラフの外縁ジャンルを把握する。
**この外縁にも登場しないジャンル** が真の対極ジャンル候補となる。

プロンプトにジャンルの指定があればそれを優先。

### 3. そのジャンルの「入口」アーティストを探す（並行実行）

以下2つを**同時に**実行し、候補プールを作る：

**① タグ上位**（メジャーな入口候補）：
```
get_tag_top_artists(tag=<対極ジャンル>)
```

**② 隠れた候補**（マニアックすぎないラインで）：
```
find_hidden_gems(min_listeners=10000, max_listeners=1000000)
```

2ソースの結果を合算し、除外リストを適用した残りを候補プールとする。
必要に応じて `get_artist_top_tags` でタグを確認し、ジャンルの代表性を検証する。
対極ジャンルに初めて触れるユーザーにとって**最も聴きやすい**ものを優先して選ぶ。

### 4. ユーザーの好みとの「意外な接点」を探す

完全に対極でも、ユーザーが引っかかれるポイントがあると推薦しやすくなる。以下を意識して候補を絞る：

- **聴感の質感**：ユーザーがロック好きなら「ライブ感のある録音」「バンドサウンド」を持つアーティスト
- **エネルギー感**：激しめが好きなら対極ジャンルでも「力強い」ものを選ぶ
- **歌詞・ボーカル**：インディ好きなら「詩的・内省的なリリック」を持つアーティスト
- **時代感**：クラシックロック好きなら「レトロな質感」のある対極ジャンルを

### 5. 除外ルール

- consistent / rising / fading に含まれるアーティストは推薦しない
- ユーザーの上位アーティストは除外

### 6. 出力フォーマット

```
## [対極ジャンル名]の世界へ

あなたの普段の音楽（[支配的ジャンル]）とは対極にある **[対極ジャンル]** から、
それでもあなたに刺さりそうな入口アーティストを選びました。

1. **[アーティスト名]**
   ジャンル: [タグ]
   → [なぜあなたに合うか。対極だが刺さりそうなポイントを1〜2文で。「〇〇好きなら△△という点で響くはず」という形で]

...（3〜5人）

---
気になる人がいたら流します。「もっとコアなのが聴きたい」「別のジャンルで」も言ってください。
```

### 7. 再生オファー

- 選ばれたら `search_artists` でURIを確認 → `play_context` で再生
- 「もっとコアなもの」→ `traverse_artist_graph(hops=2)` でより深いゾーンへ
- 「別のジャンルで」→ 別の対極ジャンルで手順2から再実行

## 注意点

- このスキルは「音楽的距離が大きいほど価値がある」。genre-adjacent と違い、ある程度の違和感は織り込み済み
- ただし「聴けない」と感じさせるアーティストは避ける。あくまで「入口」として聴きやすいものを選ぶ
- ユーザーが「やっぱり違った」と言ったら、素直に genre-adjacent や recommended-musicians に切り替えることを提案する

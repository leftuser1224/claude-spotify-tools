# search-tracks

ユーザーのプロンプトから検索意図を読み取り、Spotifyで曲を検索して結果一覧を提示する。再生はせず、情報提示にとどめる。

## 手順

### 1. 検索クエリを組み立てる

- プロンプトから曲名・アーティスト名・キーワードを抽出する
- 「〇〇の曲」→ `artist:〇〇`、「〇〇という曲」→ `track:〇〇` のように Spotify の検索構文を活用してもよい
- 日本語アーティスト・曲名はそのまま日本語で検索する（Spotify は日本語対応）
- ヒットが少ない場合は英語表記や読み方でも再検索する
- 曲名が曖昧・断片的（「あの歌詞なんだっけ」「CM で流れてた曲」など）な場合は先に Web 検索で曲名を特定してから Spotify 検索する

### 1.5. Web 検索で曲名を特定する（必要な場合のみ）

クエリが断片的で Spotify 検索が難しい場合：

1. `WebSearch` で `"[キーワード] 曲名 アーティスト"` などを検索して正式な曲名・アーティスト名を調べる
2. 判明した正式名称で Spotify 検索に進む
3. Web 検索で見つけた情報（曲の説明など）は結果表示時に補足として添える

### 2. 検索実行

- `search_tracks(query, limit=5)` を実行
- アーティストが明確な場合は `"曲名 アーティスト名"` の形式で検索すると精度が上がる

### 3. 出力フォーマット

```
## 検索結果：「[クエリ]」

1. **[曲名]** — [アーティスト名]（[アルバム名]、[年]）
2. **[曲名]** — [アーティスト名]（[アルバム名]、[年]）
...

再生しますか？
```

- リリース年は `release_date` の先頭4文字を使う
- 結果が0件や明らかにズレている場合は別クエリで再検索する
- 最大5件まで表示。それ以上は「もっと見る？」と聞く

### 4. 再生オファー

- ユーザーが「流して」「再生して」と言ったら `search-and-play` スキルに移行するか、`play_track(uri)` で直接再生する

## 注意

- このスキルは**検索と情報提示**が目的。再生まで求められている場合は `search-and-play` を使う
- アルバム名が長い場合は省略してよい

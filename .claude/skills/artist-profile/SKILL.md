# artist-profile

アーティスト名が与えられたら、Last.fm × Spotify の両方から情報を集めて「このアーティストとは何者か」を立体的に提示する。単なるWikipedia的な紹介ではなく、音楽的な文脈（ジャンル・規模感・類似アーティスト・ディスコグラフィ）まで含めて伝えることが目的。

## 手順

### 1. データ収集（並行実行）

以下を同時に取得する：

- `get_artist_info(artist_name)` — バイオ・月間リスナー数・再生数・タグ
- `get_artist_top_tags(artist_name)` — コミュニティによる詳細ジャンルタグ
- `get_similar_artists(artist_name, limit=8)` — 類似アーティスト
- `search_artists(artist_name, limit=1)` — SpotifyのアーティストURI取得
- `search_tracks("[artist_name] [代表的な曲名 or 空白]", limit=5)` — 代表曲を探す

Spotifyのアーティストが取得できたら `get_artist_albums(artist_id, limit=8)` でディスコグラフィも取得。

### 2. 規模感の判断

Last.fm の `listeners` 数から規模感を判断して明示する：

| リスナー数 | 規模感 |
| -------- | ----------- |
| 〜5万 | アンダーグラウンド / インディー黎明期 |
| 5万〜50万 | インディーシーン / 国内認知帯 |
| 50万〜500万 | 中規模 / 国際的認知あり |
| 500万〜 | メジャー / グローバル級 |

### 3. ジャンルの読み取り

`get_artist_top_tags` のタグを以下のように解釈する：

- count が50以上のタグ = コア定義（そのアーティストの本質）
- count が10〜50 = サブ要素（影響やスタイルの一部）
- count が10未満 = 一部リスナーの個人的分類（参考程度）

地域タグ（japanese / british / irish等）と年代タグ（90s / 2020s等）があれば文脈として付加する。

### 4. 出力フォーマット

```text
## [アーティスト名]

**[規模感ラベル]** — Last.fm リスナー: [数]人 / 総再生: [数]回

### 基本情報
[バイオ要約。2〜4文。活動開始・出身・メンバー構成など。データにない場合はタグと曲名から推測]

### ジャンル
[コアタグを中心に、地域・年代・テクスチャを含めて1〜2文で言語化]
タグ: [tag1]、[tag2]、[tag3]...

### 代表曲
1. **[曲名]** — [アルバム名]（[年]）
2. **[曲名]** — [アルバム名]
3. ...（3〜5曲）

### ディスコグラフィ（主なアルバム）
- [アルバム名]（[年]）
- ...（最大5枚程度）

### 類似アーティスト
[match スコア順に3〜5組。一言コメントを添えて]
- **[名前]**（match: XX%）— [類似している点]
- ...

---
流しますか？アーティスト全体 / 特定のアルバム / 代表曲 から選べます。
```text

### 5. 再生オファーへの対応

- 「流して」→ `play_context(spotify:artist:xxx)` でアーティスト再生
- アルバム名が指定されたら `search_albums` でURIを探して `play_context`
- 曲名が指定されたら `search_tracks` で探して `play_track`

## ポイント

- バイオが英語の場合は日本語で要約して伝える
- データが少ない無名アーティストでも、タグと類似アーティストから「どんな音か」を推測して伝える
- 類似アーティストはそのまま並べるのではなく「〇〇に似ている」「〇〇の系譜」のように文脈を付ける
- プロンプトに「〇〇みたいな感じ？」などの比較があればそれに答える
- 調べたアーティストがユーザーの聴取履歴にある場合は「あなたがよく聴くアーティストです」と一言添える

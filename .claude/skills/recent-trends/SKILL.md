# recent-trends

直近4週間の聴取データを分析し、「今どんな音楽の時期か」を把握する。単なるランキングではなく、長期トレンドからの変化・台頭・後退を読み取ることが目的。

## 手順

### 1. データ収集（並行実行）

以下3つを常に取得する：

- `analyze_listening_evolution()` — 短期/中期/長期のジャンル変遷とアーティストの rising/fading/consistent を取得
- `get_top_tracks(time_range="short", limit=20)` — 直近の曲ランキング
- `get_top_artists(time_range="short", limit=20)` — 直近のアーティストランキング

> `get_recently_played(limit=50)` は **呼ばない**。レスポンスが非常に大きく（100KB超）コンテキストを圧迫するため。ユーザーから「直近の再生パターンを詳しく見たい」「今日何を聴いたか知りたい」と明示的に求められた場合のみ呼び出す。

### 2. 読み取りポイント

データから以下を読み取る：

**ジャンルシフト**
- `genre_by_period` の short vs long を比較し、増減が5pt以上あるものをピックアップ
- 増えているジャンル = 今の気分・フェーズ
- 減っているジャンル = 離れつつある傾向

**アーティストの動き**
- `rising` = 最近発見または急浮上したアーティスト → 今一番アツい人
- `fading` = 長期では聴いていたが直近は減っている → 一段落した存在
- `consistent` = ずっと聴き続けている核 → 揺るぎない好み

**直近のホット曲**
- `get_top_tracks(short)` のランキング上位からアーティストの傾向を読み取る
- 同一アーティストが複数曲ランクインしていれば「今ハマっている」サイン

### 3. 出力フォーマット

```
## 最近の音楽傾向

**今の時期を一言で：**
[ジャンルの変化や雰囲気を1文で表現 例：「UKインディーからJロックへの移行期」]

### ジャンルの変化
| ジャンル | 長期 | 短期 | 傾向 |
|---|---|---|---|
| [主要タグ] | XX% | XX% | ↑/↓/→ |
...（変化が大きいもの上位3〜5件）

### アーティストの動き
**今アツい（Rising）**
- [名前] — [一言：なぜ注目か]
...

**最近減っている（Fading）**
- [名前]、[名前]...（列挙のみでOK）

**ずっと好き（Consistent）**
- [名前]、[名前]...

### 直近でよく聴いている曲
1. [曲名] / [アーティスト]
2. ...（上位5曲程度）
```

### 4. 補足アクション

- Rising アーティストに未聴アルバムがあれば「流しますか？」と提案
- ジャンルシフトが顕著なら `find_hidden_gems` や `genre_bridge` で関連アーティストを探すことも提案できる

## ポイント

- 数字の羅列ではなく「今この人はどんな音楽の時期にいるか」を語るように解釈する
- 変化がない場合（ほぼ同じ傾向）も正直に「安定している」と伝える
- プロンプトに「最近〇〇系が多い気がする」などの仮説があれば、データで検証する

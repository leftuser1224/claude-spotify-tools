# recommended-playlists

ユーザーの好みと会話の文脈から最適な検索クエリを組み立て、プレイリストを推薦する。

## 手順

1. **データ収集**（並行して実行）
   - `get_top_artists(time_range="long", limit=10)` — 好きなジャンル・アーティスト傾向
   - `get_top_tracks(time_range="medium", limit=10)` — 最近のムード把握
   - `get_recently_played(limit=20)` — 直近の流れ

2. **プロンプトからキーワードを抽出**
   - シチュエーション（作業・ドライブ・睡眠・パーティー等）
   - 気分・テンション（落ち着いた・テンション上げたい・エモい等）
   - ジャンル・地域（UK・J-POP・ジャズ等）

3. **検索クエリを組み立てて検索**
   - プロンプトの意図 × 履歴の趣味 → 英語の検索クエリを2〜3パターン生成
   - 各クエリで `search_playlists(limit=3)` を実行
   - nullを除外し、重複を排除して候補を集める

4. **出力フォーマット**

```
## おすすめプレイリスト

1. **[プレイリスト名]**（[曲数]曲）
   → [なぜこれか：シチュエーション・気分との一致を1文で]

2. ...（3〜4個）

流しますか？
```

5. **再生**
   - ユーザーが選んだら `play_context(uri)` で再生

## 検索クエリのコツ
- Spotifyの公式プレイリストにヒットしやすいキーワードを使う（例：「lofi hip hop」「chill vibes」「indie rock workout」）
- 日本語より英語クエリの方が結果が豊富
- 年代・ジャンル・シチュエーションを組み合わせると精度が上がる

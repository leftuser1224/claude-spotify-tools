# artist-deep-dive

アーティストのディスコグラフィをロードマップ化して「今日から聴き始めるなら」の道筋を示す。情報の羅列ではなく「どれから入って、どう深めるか」の案内人になることが目的。

## 手順

### 1. データ収集（並行実行）

以下を同時に取得する：

- `search_artists(artist_name, limit=1)` — Spotify の artist_id 取得
- `get_artist_info(artist_name)` — バイオ・リスナー数・タグ
- `get_artist_top_tags(artist_name)` — ジャンルタグ

artist_id が取れたら `get_artist_albums(artist_id, limit=20)` でディスコグラフィを取得する。

### 2. ディスコグラフィを整理する

取得したアルバムを以下の3層に分類する：

| 層 | 目安 | 意味 |
|---|---|---|
| **入門** | 1〜2枚 | 最もアクセスしやすく、アーティストの核が詰まっている作品 |
| **代表作** | 2〜3枚 | そのアーティストを語るうえで外せない定番 |
| **深掘り** | 2〜3枚 | 実験的・初期作・後期・レア盤など、ファン向け |

分類の根拠は以下から判断する：
- リリース年（初期 vs 後期）
- アルバム名の知名度（タイトルが文化的に有名かどうか）
- `get_artist_info` のバイオや `get_artist_top_tags` のコンテキスト
- シングル・コンピは原則除外し、スタジオアルバムを優先

### 3. 主要アルバムの詳細を補完する（必要に応じて）

入門盤・代表作の中で「説明が必要なもの」があれば `get_album_info(artist_name, album_name)` を呼んで収録曲・タグ・再生数を補完する。全アルバムに呼ぶ必要はない（2〜3枚に絞る）。

### 4. 出力フォーマット

```
## [アーティスト名] 聴取ロードマップ

**[ジャンル] / [規模感] — Last.fm リスナー: [数]人**
[アーティストの本質を1〜2文で。バイオから引用せず自分の言葉で]

---

### まず聴くべき（入門）
**[アルバム名]**（[年]）
→ [なぜここから入るべきか。アーティストのどの側面が最も凝縮されているか]

**[アルバム名]**（[年]）— あれば2枚目
→ [...]

---

### 次に行くべき（代表作）
- **[アルバム名]**（[年]）— [一言：何が特別か]
- **[アルバム名]**（[年]）— [一言]
- ...

---

### 深掘り（ファン向け）
- **[アルバム名]**（[年]）— [一言：どんな人向けか]
- ...

---

どこから聴きますか？
- **入門盤から** → すぐ再生します
- **「This Is [アーティスト名]」** → Spotify公式プレイリストで全体像をつかむ
- **アルバム名を指定** → そのアルバムをかけます
```

### 5. 再生への対応

ユーザーの返答に応じて：

- **入門盤・アルバム指定** → `search_albums(query="[アーティスト名] [アルバム名]", limit=3)` でURIを取得して `play_context`
- **「This Is」プレイリスト** → `get_this_is_playlist(artist_name)` でURIを取得して `play_context`
- **アーティスト全体** → `search_artists` で取得した URI で `play_context(spotify:artist:xxx)`

## ポイント

- 分類に迷ったら「このアーティストを全く知らない人が最初に聴くとしたら」を基準にする
- 入門盤は必ずしも最高傑作でなくていい。聴きやすさ・わかりやすさを優先する
- アルバム数が少ない（3枚以下）アーティストはロードマップより「全作品コメント付き一覧」形式にする
- 活動歴が長いアーティストは時代ごとに色が違うことを伝える（例：初期 vs 後期でジャンルが変わっている等）
- バイオが英語でも出力は必ず日本語にする

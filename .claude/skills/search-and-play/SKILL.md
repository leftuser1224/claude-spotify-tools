# search-and-play

ユーザーのプロンプトから曲・アーティストを特定し、Spotifyで検索してすぐに再生する。ワンアクションで「探して流す」を完結させるのが目的。

## 手順

### 1. 検索クエリを組み立てる

- プロンプトから曲名・アーティスト名・キーワードを抽出する
- 「〇〇の曲」→ アーティスト検索も視野に入れる
- 曲名が明確なら `"曲名 アーティスト名"` の形式で検索すると精度が上がる
- 日本語のままで検索してよい（Spotify は日本語対応）
- 曲名が曖昧・断片的（「あのサビの曲」「CM で流れてたやつ」など）な場合は先に Web 検索で特定する

### 1.5. Web 検索で曲名を特定する（必要な場合のみ）

クエリが断片的で Spotify 検索が難しい場合：

1. `WebSearch` で `"[キーワード] 曲名 アーティスト"` などを検索して正式な曲名・アーティスト名を調べる
2. 判明した正式名称で Spotify 検索に進む
3. 再生後に「〇〇の曲ですね！」と一言添えると親切

### 2. 検索実行

- `search_tracks(query, limit=3)` を実行して候補を絞る
- 1件目が明らかに正解であれば即再生
- 複数候補がある場合や曖昧な場合は候補を提示してから再生

### 3. 再生

- `play_track(uri)` で再生する
- アーティスト全体をかけてほしい場合は `play_context(spotify:artist:xxx)` を使う
- アルバムを再生したい場合は `play_context(spotify:album:xxx)` を使う

### 4. 再生後の報告

再生後に一言報告する：

```
**[曲名]** / [アーティスト名] を再生しました！
```

## 判断フロー

```
曲名が明確？
  ├─ Yes → search_tracks → 1位が正解なら即 play_track
  └─ No（「〇〇っぽい曲」「〇〇系」など） → vibe-playlist や recommended-playlist を検討
```

## 注意

- 曲が見つからない場合はクエリを変えて再検索（英語表記、読み方など）
- 「〇〇のアルバムをかけて」→ `search_albums` で探して `play_context`
- 「〇〇をシャッフルで」→ 再生前に `set_shuffle(true)` を呼ぶ
- このスキルは**再生まで完結させる**のが責務。検索だけでよい場合は `search-tracks` を使う

# discovery-session

現在の「開拓度スコア」を診断し、今の聴取傾向から自然につながる未知のアーティストを提案する。単なるレコメンドではなく「なぜこの人か」の文脈を添えて提示することが目的。

## 手順

### 1. 現状診断（並行実行）

以下2つを同時取得する：

- `analyze_discovery_rate()` — 発見スコア・リスナータイプ・直近の新着アーティストリスト
- `get_top_artists(time_range="short", limit=10)` — 現在のトップアーティスト（seed選定に使う）

### 2. スコアに応じた発掘戦略を選ぶ

`discovery_score` の値で戦略を切り替える：

| スコア | タイプ | 戦略 |
|---|---|---|
| 70以上 | 冒険家 | `find_hidden_gems(min_listeners=10000, max_listeners=200000)` でよりマイナーな宝探し |
| 40〜69 | バランス型 | `find_hidden_gems(min_listeners=50000, max_listeners=500000)` で安定した新規開拓 |
| 40未満 | 快適ゾーン型 | `find_hidden_gems(min_listeners=100000, max_listeners=1000000)` で馴染みやすい入口から |

- 戦略決定後、上記の `find_hidden_gems` を実行する
- さらに `get_top_artists(short)` の上位2アーティストを seed に `get_similar_artists()` を並行実行し、未聴の類似アーティストを補足候補として取得する

### 3. 推薦アーティストを選定する

以下の優先順位で最終推薦5〜7人を選ぶ：

1. `find_hidden_gems` の結果（リスナー数・類似度で上位のもの）
2. `get_similar_artists` の結果から `analyze_discovery_rate` の `consistent_favorites` に含まれない人
3. `analyze_discovery_rate` の `new_artists_recently`（最近自然に発見した人）— すでに知っているが定着前の存在として紹介

### 4. 出力フォーマット

```
## 発見セッション

**開拓度スコア：XX / 100（[リスナータイプ]）**
[タイプの説明を1文で。例：「信頼できるアーティストを繰り返し聴く傾向があります。」]

### 今回おすすめの新規アーティスト

**[アーティスト名]**
- どこから来た：[seed アーティスト名] の類似 / [ジャンル] の隠れた存在
- 一言：[音楽性・雰囲気を1〜2文で]

...（5〜7人）

---
プレイリストにして聴きますか？　アーティストを絞っても構いません。
```

### 5. プレイリスト化（ユーザーが希望した場合）

「聴く」「プレイリストにして」など再生を求められたら：

1. 推薦アーティストに対して `search_tracks(query="artist:{名前}", limit=3)` を **並行実行**
2. 各アーティストから2曲ずつ選んで URI を収集
3. `create_playlist(name="発見セッション: {日付}", description="discovery-session で発掘したアーティスト集")` で作成
4. `add_tracks_to_playlist` で一括追加
5. `play_context` で再生開始

## ポイント

- スコアの良し悪しは価値判断しない。快適ゾーン型も「信頼できる音楽との深いつながり」として肯定的に伝える
- 推薦時は「このアーティストから自然につながる人」という文脈を必ず添える。脈絡のないリストにしない
- `new_artists_recently` に含まれるアーティストは「すでに耳にしているが、まだ深掘りしていない人」として紹介できる
